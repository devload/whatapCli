# 브라우저 RUM 분석 가이드

이 문서는 브라우저 Real User Monitoring 데이터를 분석하여 성능 문제를 진단하고 해결하는 방법을 설명합니다.

---

## 1. 이상 징후 감지

### 1.1 모니터링해야 할 핵심 지표

| 지표 | 정상 범위 | 주의 | 심각 |
|------|----------|------|------|
| 페이지 로드 시간 | < 3초 | 3-5초 | > 5초 |
| TTFB (Time to First Byte) | < 200ms | 200-500ms | > 500ms |
| AJAX 응답시간 | < 300ms | 300-1000ms | > 1초 |
| AJAX 에러율 | < 0.1% | 0.1-1% | > 1% |
| JS 에러 발생률 | < 0.5% | 0.5-2% | > 2% |
| 리소스 로딩 실패율 | < 0.1% | 0.1-0.5% | > 0.5% |

### 1.2 이상 징후 패턴

**패턴 A: 특정 페이지만 느림**
```
증상: /checkout 페이지만 로딩이 8초
의심: 해당 페이지 특화 문제 (API 호출, 리소스, 렌더링)
```

**패턴 B: 특정 시간대 전체 느림**
```
증상: 매일 오후 2시 전체 페이지 느려짐
의심: 백엔드 부하, DB 부하, 외부 API 지연
```

**패턴 C: 특정 브라우저만 에러**
```
증상: Safari에서만 JS 에러 다수 발생
의심: 브라우저 호환性问题, 폴리필 누락
```

**패턴 D: 모바일만 느림**
```
증상: 데스크톱은 2초, 모바일은 6초
의심: 리소스 크기, 3G/4G 환경, 모바일 최적화 부재
```

## 2. 원인 분석 플로우

### 2.1 페이지 로딩 느림 → 원인 찾기

```
페이지 로딩 느림
       │
       ▼
   TTFB가 느린가?
       │
   ┌───┴───┐
   YES     NO
   │       │
   ▼       ▼
백엔드   Frontend가 느린가?
문제         │
         ┌───┴───┐
         YES     NO
         │       │
         ▼       ▼
     JS 실행   다운로드가 느린가?
     최적화        │
               ┌───┴───┐
               YES     NO
               │       │
               ▼       ▼
           리소스    렌더링
           최적화    최적화
```

**분석 명령어:**

```bash
# 1. 페이지 로드 타이밍 분해
$ whatap step pageload --pcode 12345 --page "/checkout" --duration 1h

  /checkout@593000 (count: 5, total: 4.04s)
  ------------------------------------------------------------
         DNS     19ms |                          ← 네트워크
     Connect     37ms |                          ← 네트워크
         SSL     56ms |                          ← 네트워크
        TTFB    320ms |##                        ← 백엔드 의심
    Download    244ms |##                        ← 리소스 크기
     Backend    435ms |###                       ← 백엔드 확정
    Frontend    3.60s |############################## ← 문제 여기!
      Render    762ms |######                     ← 렌더링

# 해석: Frontend(3.6초)가 전체 시간의 89% 차지
# → JS 실행, 리소스 로딩, 렌더링 문제
```

```bash
# 2. 해당 페이지의 AJAX 확인
$ whatap trace /checkout --pcode 12345 --only ajax --slow 500

=== AJAX Requests (>500ms) ===
+-----------------------------------------+-------+----------+
| URL                                     | Count | Duration |
+-----------------------------------------+-------+----------+
| /api/validate-coupon                    | 45    | 2.3s     |  ← 의심
| /api/shipping-options                   | 45    | 1.8s     |
+-----------------------------------------+-------+----------+

# 해석: /api/validate-coupon이 2.3초 소요
# → 백엔드 API 문제이거나 요청이 너무 많음
```

```bash
# 3. 리소스 로딩 확인
$ whatap trace /checkout --pcode 12345 --only resources --slow 1000

=== Resources (>1s) ===
+--------+------------------------------------------+----------+--------+
| Type   | URL                                      | Duration | Size   |
+--------+------------------------------------------+----------+--------+
| script | cdn.shop.com/bundle.js                   | 2.1s     | 4.5MB  | ← 문제
| img    | cdn.shop.com/hero-banner.jpg             | 1.5s     | 8MB    |
+--------+------------------------------------------+----------+--------+

# 해석: bundle.js 4.5MB 다운로드에 2.1초
# → 번들 사이즈 최적화 필요
```

### 2.2 JS 에러 급증 → 원인 찾기

```
JS 에러 급증
     │
     ▼
특정 에러 타입인가?
     │
 ┌───┴───────────┐
 TypeError      ReferenceError
     │              │
     ▼              ▼
 undefined      변수/함수
 접근           미정의
     │
     ▼
API 응답 구조
변경?
```

**분석 명령어:**

```bash
# 1. 에러 유형 분포
$ whatap step errors --pcode 12345 --duration 1h

+------------+------------------------------------------+-------+
| Error Type | Message                                  | Count |
+------------+------------------------------------------+-------+
| TypeError  | Cannot read property 'id' of undefined   | 234   |
| SyntaxError| Unexpected token '<'                     | 89    |
| fetchError | NetworkError: Failed to fetch            | 45    |
+------------+------------------------------------------+-------+

# 해석:
# - TypeError 다수 → API 응답에 기대한 필드가 없음
# - SyntaxError + '<' → API가 JSON 대신 HTML 에러 페이지 반환
# - fetchError → 네트워크/CORS 문제
```

```bash
# 2. 에러 발생 페이지 확인
$ whatap step errors --pcode 12345 --type TypeError --duration 1h

+------------+----------------------------------------+---------+
| Key        | Message                                | Browser |
+------------+----------------------------------------+---------+
| /cart@0    | Cannot read property 'id' of undefined | Chrome  |
| /cart@0    | Cannot read property 'id' of undefined | Safari  |
+------------+----------------------------------------+---------+

# 해석: /cart 페이지에서 집중 발생
```

```bash
# 3. 해당 페이지 AJAX 확인 (API 응답 문제?)
$ whatap trace /cart --pcode 12345 --only ajax

=== AJAX Requests ===
+-----------------------------------------+-------------+
| URL                                     | Errors      |
+-----------------------------------------+-------------+
| /api/cart/items                         | 5xx:45 4xx:0| ← 문제
+-----------------------------------------+-------------+

# 해석: /api/cart/items 500 에러 45건
# → 백엔드 장애로 인해 JS에서 undefined 처리 실패
```

### 2.3 AJAX 에러율 급증 → 원인 찾기

```
AJAX 에러율 급증
      │
      ▼
  에러 코드 확인
      │
  ┌───┼────────┐
  5xx 4xx     Network
  │   │        │
  ▼   ▼        ▼
백엔드 권한/  CORS/
장애  요청오류 네트워크
```

**분석 명령어:**

```bash
# 1. AJAX 에러 현황
$ whatap step ajax --pcode 12345 --errors --duration 1h

+-----------------------------------------+-------+----------+-------------+
| URL                                     | Count | Duration | Errors      |
+-----------------------------------------+-------+----------+-------------+
| /api/checkout/process                   | 234   | 3.2s     | 5xx:189 4xx:0|
| /api/user/profile                       | 567   | 1.1s     | 5xx:0 4xx:45 |
+-----------------------------------------+-------+----------+-------------+

# 해석:
# - /api/checkout/process 5xx 다수 → 백엔드 장애
# - /api/user/profile 4xx 다수 → 인증/권한 문제
```

```bash
# 2. 5xx 에러 페이지 추가 분석
$ whatap trace /checkout --pcode 12345 --only errors

=== Errors ===
+------------+------------------------------------------+-------+
| Error Type | Message                                  | Count |
+------------+------------------------------------------+-------+
| fetchError | Internal Server Error                    | 189   |
+------------+------------------------------------------+-------+

# 해석: 백엔드 500 에러가 프론트엔드에 전파됨
# → 백엔드 로그 확인 필요
```

## 3. 실제 트러블슈팅 케이스

### 케이스 1: 결제 페이지 이탈률 급증

**증상:**
- 결제 완료율이 65% → 45%로 하락
- CS 문의 증가: "결제가 안 돼요"

**분석 과정:**

```bash
# Step 1: 결제 페이지 로딩 확인
$ whatap step pageload --pcode 12345 --page "/checkout" --duration 24h

  /checkout@593000 (count: 1,234, total: 6.8s)  ← 6.8초? 너무 느림
  ------------------------------------------------------------
        TTFB    1.2s  |##########
    Download    2.3s  |####################
     Frontend    3.3s |#############################

# Step 2: 느린 AJAX 찾기
$ whatap trace /checkout --pcode 12345 --only ajax --slow 1000

+-----------------------------------------+-------+----------+
| /api/payment/validate                   | 1,234 | 3.5s     | ← 여기!
+-----------------------------------------+-------+----------+

# Step 3: 백엔드 팀에 전달 → 결제 PG사 응답 지연 확인
```

**원인:** 결제 검증 API가 외부 PG사 타임아웃으로 3.5초 대기

**해결:**
- PG사 타임아웃 단축 (30초 → 5초)
- 타임아웃 시 대체 결제수단 노출

### 케이스 2: 모바일에서만 앱 다운로드 버튼 작동 안 함

**증상:**
- 모바일 사용자 "다운로드 버튼 누르면 아무 반응 없음"

**분석 과정:**

```bash
# Step 1: 모바일 에러 확인
$ whatap step errors --pcode 12345 --duration 1h

+------------+------------------------------------------+---------+
| Error Type | Message                                  | Device  |
+------------+------------------------------------------+---------+
| TypeError  | window.webkit is undefined               | Mobile  | ← iOS
| TypeError  | AndroidInterface is not defined          | Mobile  | ← Android
+------------+------------------------------------------+---------+

# Step 2: 해당 페이지 찾기
$ whatap step errors --pcode 12345 --type TypeError --duration 1h

+----------------+---------+-------------------+
| Key            | Browser | Device            |
+----------------+---------+-------------------+
| /download@0    | Safari  | Mobile (iPhone)   |
| /download@0    | Chrome  | Mobile (Android)  |
+----------------+---------+-------------------+
```

**원인:** 앱 딥링크 호출 시 네이티브 브릭지 객체 존재 여부 체크 누락

**해결:**
```javascript
// Before
window.webkit.messageHandlers.download.postMessage(...)

// After
if (window.webkit?.messageHandlers?.download) {
  window.webkit.messageHandlers.download.postMessage(...)
} else {
  // 웹 다운로드 폴백
}
```

### 케이스 3: 새벽 2시마다 사이트 느려짐

**증상:**
- 매일 새벽 2시경 페이지 로딩 10초 이상

**분석 과정:**

```bash
# Step 1: 시간대별 응답시간 확인
$ whatap stat query --pcode 12345 --category rum_page_load --field avg_duration --duration 7d

time                  value
...
2024-01-15 01:50      2.3s
2024-01-15 02:00      12.5s  ← 문제 발생
2024-01-15 02:10      11.8s
2024-01-15 02:30      2.5s

# Step 2: 해당 시간대 리소스 확인
$ whatap step resources --pcode 12345 --duration 30m --stime 1705278000000

+--------+------------------------------------------+----------+
| script | cdn.analytics.com/tracker.js             | 8.5s     | ← 의심
+--------+------------------------------------------+----------+

# Step 3: 외부 스크립트 확인 → 분석 도구 새벽 2시 대규모 배치
```

**원인:** 외부 분석 스크립트가 새벽 2시 배치 작업으로 응답 지연

**해결:**
- 분석 스크립트 async/defer 로드
- 타임아웃 설정 (3초 후 스크립트 로드 실패해도 페이지 동작)

## 4. 핵심 분석 체크리스트

### 페이지 로딩 느릴 때

- [ ] TTFB > 500ms? → 백엔드 의심
- [ ] Frontend > 전체의 50%? → JS/리소스 문제
- [ ] 특정 리소스 > 1MB? → 최적화 필요
- [ ] 3rd party 스크립트 > 3개? → 영향도 확인
- [ ] 이미지 > 500KB? → 압축/지연로드

### JS 에러 급증할 때

- [ ] 특정 에러 타입 집중? → 패턴 분석
- [ ] 특정 브라우저만? → 호환성 문제
- [ ] 특정 페이지만? → 해당 기능 집중 분석
- [ ] 5xx 에러 동반? → 백엔드 장애 연관
- [ ] 새 배포 후 발생? → 릴리스 회귀

### AJAX 에러 급증할 때

- [ ] 5xx 비율 > 1%? → 백엔드 장애
- [ ] 4xx 비율 > 1%? → 요청/인증 문제
- [ ] Network 에러? → CORS/연결 문제
- [ ] Timeout 에러? → 응답시간 최적화
- [ ] 특정 API만? → 해당 API 집중 분석

```bash
$ whatap step resources --pcode 12345 --slow 1000 --duration 1h
```
