# WhatAp CLI - 브라우저 RUM 분석 예시

브라우저 Real User Monitoring(RUM) 데이터 분석 시나리오와 CLI 사용 예시입니다.

## 사전 요구사항

```bash
# 로그인 (이메일/비밀번호 필요 - MXQL/Step은 웹 세션 필요)
whatap login -e user@example.com -p 'password'

# 프로젝트 확인
whatap projects --filter BROWSER
```

---

## 1. Step 명령어 - 개별 데이터 조회

### 1.1 페이지 로드 분석

```bash
# 전체 페이지 로드 현황
whatap step pageload --pcode 12345 --duration 1h

# 느린 페이지만 (3초 이상)
whatap step pageload --pcode 12345 --slow 3000 --duration 1h

# 특정 페이지만
whatap step pageload --pcode 12345 --page "/checkout" --duration 1h

# JSON 출력
whatap step pageload --pcode 12345 --duration 1h --json
```

**출력 예시:**

```bash
$ whatap step pageload --pcode 27506 --duration 1h
```

## 2. Trace 명령어 - 연관 데이터 분석

### 2.1 기본 사용법

```bash
# 1단계: Key 확인
whatap step pageload --pcode 12345 --duration 1h

# 2단계: 해당 페이지의 모든 연관 데이터 조회
whatap trace /checkout@463000 --pcode 12345
```

### 2.2 Trace 옵션

```bash
# 요약만
whatap trace /products --pcode 12345 --summary

# 특정 카테고리만
whatap trace /products --pcode 12345 --only ajax
whatap trace /products --pcode 12345 --only errors
whatap trace /products --pcode 12345 --only resources
whatap trace /products --pcode 12345 --only pageload

# 느린 항목만 (>2초)
whatap trace /products --pcode 12345 --slow 2000

# JSON 출력
whatap trace /products --pcode 12345 --json

# CSV 출력
whatap trace /products --pcode 12345 --csv

# 시간 범위 지정
whatap trace /products --pcode 12345 --duration 30m
```

## 3. 분석 시나리오

### 시나리오 1: 페이지 로드 지연 원인 분석

```bash
# 1단계: 느린 페이지 찾기
whatap step pageload --pcode 12345 --slow 3000 --duration 1h

# 2단계: 해당 페이지 Trace
whatap trace /checkout@463000 --pcode 12345 --duration 1h

# 3단계: 느린 AJAX/리소스 확인
whatap trace /checkout@463000 --pcode 12345 --only ajax --slow 1000
whatap trace /checkout@463000 --pcode 12345 --only resources --slow 500
```

**분석 포인트:**
- Backend vs Frontend 비중
- 느린 AJAX 요청
- 큰 리소스 (이미지, 번들)
- 3rd party 스크립트

### 시나리오 2: JavaScript 에러 원인 추적

```bash
# 1단계: 에러 목록 확인
whatap step errors --pcode 12345 --duration 1h

# 2단계: 특정 에러 타입 분석
whatap step errors --pcode 12345 --type TypeError --duration 1h

# 3단계: 해당 페이지의 연관 데이터
whatap trace /dashboard@123456 --pcode 12345 --duration 1h

# 4단계: 에러와 AJAX 실패 연관성 확인
whatap trace /dashboard@123456 --pcode 12345 --only ajax --only errors
```

### 시나리오 3: AJAX 실패 분석

```bash
# 1단계: 에러가 있는 AJAX 찾기
whatap step ajax --pcode 12345 --errors --duration 1h

# 2단계: 해당 페이지 Trace
whatap trace /account@473000 --pcode 12345

# 3단계: JSON으로 상세 분석
whatap trace /account@473000 --pcode 12345 --only ajax --json
```

### 시나리오 4: 3rd Party 스크립트 영향 분석

```bash
# 1단계: 리소스 로딩 확인
whatap step resources --pcode 12345 --duration 1h

# 2단계: 느린 리소스만
whatap trace /products --pcode 12345 --only resources --slow 1000

# 3단계: CSV로 내보내기
whatap trace /products --pcode 12345 --only resources --csv > resources.csv
```

### 시나리오 5: 전환율 저하 원인 분석

```bash
# 1단계: 결제 페이지 분석
whatap trace /checkout --pcode 12345 --summary

# 2단계: 에러 확인
whatap step errors --pcode 12345 --page "/checkout" --duration 24h

# 3단계: AJAX 실패 확인
whatap step ajax --pcode 12345 --page "/checkout" --errors --duration 24h

# 4단계: 느린 로딩 확인
whatap step pageload --pcode 12345 --page "/checkout" --slow 3000 --duration 24h
```

## 4. CI/CD 연동

### 일일 리포트 생성

```bash
#!/bin/bash
# daily_browser_report.sh

PCODE=12345
DATE=$(date +%Y%m%d)
OUTPUT_DIR="./reports/$DATE"

mkdir -p $OUTPUT_DIR

# 페이지 로드 리포트
whatap step pageload --pcode $PCODE --duration 24h --json > $OUTPUT_DIR/pageload.json

# 에러 리포트
whatap step errors --pcode $PCODE --duration 24h --json > $OUTPUT_DIR/errors.json

# AJAX 에러 리포트
whatap step ajax --pcode $PCODE --errors --duration 24h --json > $OUTPUT_DIR/ajax_errors.json

echo "Report generated: $OUTPUT_DIR"
```

### Slack 알림

```bash
#!/bin/bash
# check_errors.sh

PCODE=12345
WEBHOOK_URL="https://hooks.slack.com/services/xxx"

# 에러 수 카운트
ERROR_COUNT=$(whatap step errors --pcode $PCODE --duration 1h --json | jq 'length')

if [ "$ERROR_COUNT" -gt 10 ]; then
  curl -X POST $WEBHOOK_URL \
    -H 'Content-Type: application/json' \
    -d "{\"text\": \"⚠️ Browser Error Alert: $ERROR_COUNT errors in the last hour\"}"
fi
```

## 5. MXQL 직접 쿼리

### 고급 쿼리 예시

```bash
# 특정 기간, 특정 페이지의 모든 데이터
whatap mxql --pcode 12345 "CATEGORY rum_page_load_each_page
TAGLOAD
SELECT [page_group, page_load_duration, page_load_count, @timestamp]
FILTER { page_group == '/checkout' }
LIMIT 100"

# 평균 응답 시간이 느린 페이지
whatap mxql --pcode 12345 "CATEGORY rum_ajax_each_page
TAGLOAD
SELECT [page_group, ajax_duration, ajax_count]
LIMIT 100"

# Core Web Vitals
whatap mxql --pcode 12345 "CATEGORY rum_web_vitals_each_page
TAGLOAD
SELECT [page_group, lcp, fid, cls]
LIMIT 50"
```

## 6. 전체 명령어 옵션

### step pageload
```
--pcode <PCODE>      프로젝트 코드
--page <PAGE>        페이지 필터
--slow <MS>          느린 페이지만 (ms)
--duration <DUR>     시간 범위 (1h, 30m, 1d)
--limit <N>          최대 결과 수
--raw                원본 JSON
```

### step ajax
```
--pcode <PCODE>      프로젝트 코드
--page <PAGE>        페이지 필터
--errors             에러가 있는 요청만
--slow <MS>          느린 요청만 (ms)
--duration <DUR>     시간 범위
--limit <N>          최대 결과 수
--raw                원본 JSON
```

### step resources
```
--pcode <PCODE>      프로젝트 코드
--page <PAGE>        페이지 필터
--type <TYPE>        리소스 타입 (script, image, link, font, xhr, fetch)
--slow <MS>          느린 리소스만 (ms)
--duration <DUR>     시간 범위
--limit <N>          최대 결과 수
--raw                원본 JSON
```

### step errors
```
--pcode <PCODE>      프로젝트 코드
--page <PAGE>        페이지 필터
--type <TYPE>        에러 타입 (TypeError, ReferenceError, etc.)
--browser <BROWSER>  브라우저 필터
--duration <DUR>     시간 범위
--limit <N>          최대 결과 수
--raw                원본 JSON
```

### trace
```
<KEY>                페이지 키 (예: /cart@123456)
--pcode <PCODE>      프로젝트 코드
--duration <DUR>     시간 범위 (기본: 1h)
--only <CAT>         특정 카테고리만 (pageload, ajax, resources, errors)
--slow <MS>          느린 항목만
--summary            요약만
--json               JSON 출력
--csv                CSV 출력
--raw                원본 API 응답
```

## 7. 문제 해결

### 데이터가 안 보일 때

1. 시간 범위 확인 (`--duration`)
2. pcode 확인
3. 로그인 상태 확인 (`whatap whoami`)
4. 웹 세션 필요 (`--api-key`로는 step/trace 불가)

### Key 형식

- `page_group@time_suffix` (예: `/checkout@463000`)
- 또는 `page_group`만 (예: `/checkout`)
- `@` 뒤 숫자는 time % 1,000,000 값

### MXQL 문법 에러

```bash
# verbose 모드로 쿼리 확인
whatap mxql --pcode 12345 "..." --verbose
```
