# WhatAp CLI - 모바일 앱 분석 예시

모바일 앱 모니터링 데이터 분석 시나리오와 CLI 사용 예시입니다.

## 사전 요구사항

```bash
# 로그인 (이메일/비밀번호)
whatap login -e user@example.com -p 'password'

# 또는 API 키로 로그인 (CI/CD)
whatap login --api-key <key> --pcode <pcode>

# 프로젝트 확인
whatap projects --filter MOBILE
```

## 1. 크래시 분석

### 1.1 크래시 목록 조회

```bash
# MXQL로 크래시 데이터 조회
whatap mxql --pcode 12345 --category mobile_crash --duration 24h

# 특정 기기에서 발생한 크래시
whatap mxql --pcode 12345 "CATEGORY mobile_crash
TAGLOAD
SELECT [crashType, crashMessage, device, osVersion, count, @timestamp]
FILTER { device == 'iPhone 14' }
LIMIT 50"
```

**출력 예시:**

```bash
$ whatap mxql --pcode 12345 --category mobile_crash --duration 24h
```
```
crashType              crashMessage                           device       osVersion   count
NullPointerException    Cannot invoke method on null object    iPhone 14     iOS 17.2    45
IndexOutOfBoundsException Index 5 out of bounds               Samsung S23   Android 14  23
IllegalStateException    Fragment not attached to activity     iPhone 13     iOS 16.6    18
NetworkException         Connection timeout                    iPad Pro      iPadOS 17   12
```

### 1.2 크래시 유형별 집계

```bash

TAGLOAD

LIMIT 100"
```

**출력 예시:**

```bash
$ whatap mxql --pcode 12345 "CATEGORY mobile_crash TAGLOAD SELECT [crashType, count] LIMIT 100"
```

## 6. 분석 시나리오

### 시나리오 1: 특정 기기 크래시 집중 분석

```bash
# 1단계: 크래시가 많이 발생한 기기 찾기
whatap mxql --pcode 12345 "CATEGORY mobile_crash
TAGLOAD
SELECT [device, count]
LIMIT 20"

# 2단계: 해당 기기의 크래시 상세
whatap mxql --pcode 12345 "CATEGORY mobile_crash
TAGLOAD
SELECT [crashType, crashMessage, osVersion, count]
FILTER { device == 'Samsung Galaxy S23' }
LIMIT 50"

# 3단계: 해당 기기의 ANR 여부 확인
whatap mxql --pcode 12345 "CATEGORY mobile_device_session
TAGLOAD
SELECT [anr_count, crash_count, osVersion]
FILTER { device == 'Samsung Galaxy S23' && crash_count > 0 }
LIMIT 50"
```

### 시나리오 2: OS 업데이트 후 문제 분석

```bash
# 1단계: 특정 OS 버전 크래시 확인
whatap mxql --pcode 12345 "CATEGORY mobile_crash
TAGLOAD
SELECT [osVersion, crashType, count]
FILTER { osVersion like 'iOS 17' }
LIMIT 100"

# 2단계: 이전 버전과 비교
whatap mxql --pcode 12345 "CATEGORY mobile_crash
TAGLOAD
SELECT [osVersion, count]
LIMIT 100"

# 3단계: 해당 OS 버전의 세션 안정성
whatap mxql --pcode 12345 "CATEGORY mobile_device_session
TAGLOAD
SELECT [osVersion, session_count, crash_count, anr_count]
FILTER { osVersion like 'iOS 17' }
LIMIT 50"
```

### 시나리오 3: 일일 크래시 리포트

```bash
# JSON으로 내보내기
whatap mxql --pcode 12345 "CATEGORY mobile_crash
TAGLOAD
SELECT [crashType, crashMessage, device, osVersion, count, @timestamp]
LIMIT 200" --json > crash_report_$(date +%Y%m%d).json

# CSV로 변환 (jq 사용)
cat crash_report_$(date +%Y%m%d).json | jq -r '.[] | [.crashType, .crashMessage, .device, .count] | @csv' > crash_report.csv
```

### CI/CD 연동

```bash
#!/bin/bash
# daily_crash_report.sh

PCODE=12345
THRESHOLD=10

# 크래시 수 조회
CRASH_COUNT=$(whatap mxql --pcode $PCODE "CATEGORY mobile_crash
TAGLOAD
SELECT [count]
LIMIT 1" --json | jq '[.[].count] | add')

echo "Today's crash count: $CRASH_COUNT"

if [ "$CRASH_COUNT" -gt "$THRESHOLD" ]; then
  echo "WARNING: Crash count exceeds threshold!"
  # Slack webhook 등으로 알림
  exit 1
fi
```

## 9. 전체 옵션

```
whatap mxql [OPTIONS] [QUERY]

Options:
  --pcode <PCODE>      프로젝트 코드
  -f, --file <FILE>    MXQL 파일에서 읽기
  --category <CAT>     카테고리만 지정 (자동 쿼리 생성)
  --stime <TIME>       시작 시간 (epoch ms)
  --etime <TIME>       종료 시간 (epoch ms)
  --limit <N>          최대 결과 수 (기본: 100)
  --json               JSON 출력

whatap spot [OPTIONS]
  --pcode <PCODE>      프로젝트 코드
  --keys <KEYS>        메트릭 키 (쉼표 구분)

whatap stat query [OPTIONS]
  --pcode <PCODE>      프로젝트 코드
  --category <CAT>     카테고리
  --field <FIELD>      필드명
  --duration <DUR>     시간 범위 (예: 1h, 30m, 1d)
```

## 10. 문제 해결

### 크래시 스택트레이스가 난독화되어 있을 때

1. ProGuard/dSYM 파일이 업로드되었는지 확인
2. 버전이 일치하는지 확인
3. 업로드 후 새 크래시는 해제되어 표시됨

### 데이터가 안 보일 때

1. 시간 범위 확인 (`--duration`, `--stime`, `--etime`)
2. pcode 확인
3. API 권한 확인 (`whatap whoami`)

### MXQL 문법 에러

```bash
# verbose 모드로 쿼리 확인
whatap mxql --pcode 12345 "..." --verbose
```
